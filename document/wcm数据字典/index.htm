TRS内容协作平台(WCM)数据字典

本数据字典包含WCM标准平台及与之一同运行的图片库,表单(不包含infogate应用), 元数据等三个选件在内的数据库表结构的说明.截止整理日期(2009-04-30),系统共有153个表. 数据字典为开发文档，仅限公司内部使用

各表详细信息 E-R对应表

在深入了解每一个表的细节前,有必要了解一下文档的说明风格。

  • 每个表的描述由主要的三部分组成:
    1. 表名:如WCMDOCUMENT表.
    2. 描述:简要说明该表存储的是什么信息
      如果在该部分看到(保留)字样,则表示该表一般情况下不为外部使用.不需要关心其内部细节
    3. 结构:以列表的方式说明表的结构.如下表所示
      表结构说明
      列名 说明 默认值 允许空值 类型 长度 备注
      DOCID 文档编号 默认值 int 100
      该表共有七列,每一行描述一个数据表字段的信息。
      1. 列名：字段的名称，图标表示该列一该表的主键列
      2. 说明：字段的作用简要描述
      3. 默认值：字段的默认取值，如果没有则留空
      4. 表示该字段允许空值，否则必须有值
      5. 类型：字段在mssql中的类型名
      6. 长度：如果该字段的类型是nvarchar，则标识该字段可容纳的字符长度；否则留空
      7. 备注：对于该列的一些补充说明，可能链接到另外一个表，也可能是链接到当前表的备注部分
  • 在表结构的说明之后，可能会有备注部分。该部分是对表结构列的一些补充
  • 表结构中的类型是以mssql类型名表示的，对应于其它数据库的类型名如下表所示
    数据类型对应表
    数据库 数据类型
    mssql int bigint smallint tinyint (n)varchar (n)char datetime (n)text
    oracle int number(38,0) number(5,0) number(5,0) varchar2 varchar2 date clob
    db2 bigint decimal(31,0) int int varchar varchar timestamp clob
    sybase int bigint smallint tinyint nvarchar nvarchar datetime text
    mysql int bigint smallint tinyint nvarchar nvarchar datetime text
  • 对于nvarchar类型的长度，其它数据库对应的长度是mssql的两倍与4000中两个值的小者
